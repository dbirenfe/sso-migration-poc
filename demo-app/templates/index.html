SSO Migration Demo — RH-SSO / RHBK Token Exchange

RH-SSO RHBK
Architecture Components

RH-SSO 7.6.5

Legacy Identity Provider (Keycloak 18.x on JBoss EAP). Runs in the rhsso namespace. Token exchange enabled as tech-preview feature.

RHBK 26.4

New Identity Provider (Keycloak 26.x on Quarkus). Runs in the rhbk namespace. Fine-grained authorization enabled for token exchange.

Token Exchange Trust

Bidirectional IdP trust configured between both Keycloaks. Each has the other registered as an Identity Provider.

Token Exchange Proxy

Transparent reverse proxy. Inspects JWT issuer, performs token exchange if needed, forwards request with correct token. Zero code changes to applications.

How It Works

1. App migrated from RH-SSO to RHBK

2. Another system still holds an RH-SSO token

3. It calls the migrated app through the proxy

4. Proxy detects issuer mismatch

5. Proxy exchanges RH-SSO token for RHBK token

6. Proxy forwards request with valid RHBK token

7. App validates token normally — no code changes

Quick Reference — Test Commands

Run these commands from any machine with curl access to the cluster. Replace variables as needed for your environment.

1. Acquire RH-SSO Token

curl -sk -X POST '{{ rhsso_external }}/auth/realms/{{ realm }}/protocol/openid-connect/token' \ -d 'client_id={{ client_id }}' \ -d 'client_secret=token-exchange-secret-12345' \ -d 'username={{ test_user }}' \ -d 'password=testpass' \ -d 'grant_type=password' \ -d 'scope=openid' | python3 -m json.tool

2. Acquire RHBK Token

curl -sk -X POST '{{ rhbk_external }}/realms/{{ realm }}/protocol/openid-connect/token' \ -d 'client_id={{ client_id }}' \ -d 'client_secret=token-exchange-secret-12345' \ -d 'username={{ test_user }}' \ -d 'password=testpass' \ -d 'grant_type=password' \ -d 'scope=openid' | python3 -m json.tool

3. Exchange RHBK Token → RH-SSO

# First get a RHBK token: RHBK_TOKEN=$(curl -sk '{{ rhbk_external }}/realms/{{ realm }}/protocol/openid-connect/token' \ -d 'client_id={{ client_id }}' -d 'client_secret=token-exchange-secret-12345' \ -d 'username={{ test_user }}' -d 'password=testpass' \ -d 'grant_type=password' -d 'scope=openid' | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])") # Exchange it at RH-SSO: curl -sk -X POST '{{ rhsso_external }}/auth/realms/{{ realm }}/protocol/openid-connect/token' \ -d 'client_id={{ client_id }}' \ -d 'client_secret=token-exchange-secret-12345' \ -d 'grant_type=urn:ietf:params:oauth:grant-type:token-exchange' \ -d "subject_token=$RHBK_TOKEN" \ -d 'subject_token_type=urn:ietf:params:oauth:token-type:access_token' \ -d 'subject_issuer=rhbk' | python3 -m json.tool

4. Exchange RH-SSO Token → RHBK

# First get an RH-SSO token: RHSSO_TOKEN=$(curl -sk '{{ rhsso_external }}/auth/realms/{{ realm }}/protocol/openid-connect/token' \ -d 'client_id={{ client_id }}' -d 'client_secret=token-exchange-secret-12345' \ -d 'username={{ test_user }}' -d 'password=testpass' \ -d 'grant_type=password' -d 'scope=openid' | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])") # Exchange it at RHBK: curl -sk -X POST '{{ rhbk_external }}/realms/{{ realm }}/protocol/openid-connect/token' \ -d 'client_id={{ client_id }}' \ -d 'client_secret=token-exchange-secret-12345' \ -d 'grant_type=urn:ietf:params:oauth:grant-type:token-exchange' \ -d "subject_token=$RHSSO_TOKEN" \ -d 'subject_token_type=urn:ietf:params:oauth:token-type:access_token' \ -d 'subject_issuer=rhsso' | python3 -m json.tool

5. Decode a JWT Token

# Decode any JWT to see its claims: echo "$TOKEN" | cut -d. -f2 | base64 -d 2>/dev/null | python3 -m json.tool

6. Check Component Status
